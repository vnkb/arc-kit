ArcKit

Transform architecture chaos into systematic, audit-ready governance.

53 AI-assisted commands that generate complete governance documents. From stakeholder analysis and risk registers to design reviews and traceability matrices—each command produces template-driven, audit-ready artifacts. Navigate discovery to delivery with built-in UK Government compliance: GDS Service Manual, Technology Code of Practice, Secure by Design.

Features

Systematic

Follow GDS Agile Delivery phases (Discovery → Alpha → Beta → Live) with structured command workflow ensuring complete, auditable project artifacts.

Compliant

Built-in compliance with Technology Code of Practice (TCoP), Secure by Design principles, GDS Service Standard, and UK Government procurement frameworks.

Auditable

Every artifact includes version control, traceability matrices, design review checklists, and comprehensive gap analysis for audit readiness.

AI-Assisted

Works with Claude Code (premier), Gemini CLI, Codex CLI, and OpenCode CLI. Claude Code provides the most complete experience with autonomous agents, automation hooks, and output validation. AI generates first drafts following templates; architects review and refine. Developed and tested on Linux; on Windows use a devcontainer or WSL2 for full hook support.

Traceable

Complete traceability chains from stakeholder goals through risks and requirements to design decisions. Every artifact links back to business drivers and outcomes.

Open Source

Free and open under MIT licence. Full transparency, community contributions welcome, and no vendor lock-in. Adapt and extend for your organisation's needs.

Multi-AI System Support

ArcKit works with four AI development environments. Claude Code is the primary development platform and provides the most complete experience.

Claude Code

Premier

The primary platform for ArcKit. Install via the ArcKit plugin for the full experience: all 53 commands, 5 autonomous research agents, 8 automation hooks, a conversational Wardley Mapping skill, and bundled MCP servers (AWS Knowledge, Microsoft Learn, Google Developer Knowledge, DataCommons).

Exclusive features:

  • Agents — 5 autonomous agents for research-heavy commands (market research, data source discovery, AWS/Azure/GCP cloud research) running in isolated context windows
  • Hooks — 8 hook scripts across 5 event types: SessionStart (version + project detection), UserPromptSubmit (project context injection + prompt secret detection), PreToolUse (ARC filename auto-correction, sensitive file protection, secret file scanning), PermissionRequest (MCP tool auto-approval), per-command Stop hooks (output validation)
  • Security hooks — Three-layer protection: prompt secret detection, sensitive file path blocking, and file content secret scanning
  • Output validation — Wardley Map math validation checks stage-evolution alignment, coordinate ranges, and OWM syntax consistency before finalizing documents
  • Knowledge compounding — Research automatically spawns reusable vendor profiles and tech notes for cross-project knowledge
  • Wardley Mapping skill — Conversational guidance with Pinecone MCP book corpus search

Installation: /plugin marketplace add tractorjuice/arc-kit

Gemini CLI

Full

Native extension with all 53 commands, templates, scripts, and bundled MCP servers (AWS Knowledge, Microsoft Learn, Google Developer Knowledge). Zero-config with auto-updates.

Limitations: No agent delegation or hooks. Research commands use inline prompts instead of isolated agents.

Extension: gemini extensions install https://github.com/tractorjuice/arckit-gemini

OpenAI Codex CLI

Core

Codex CLI for GPT-4. Commands and templates copied into project via arckit init.

Limitations: No agents, hooks, or bundled MCP servers. Manual reinstall for updates.

Installation: arckit init my-project --ai codex

OpenCode CLI

Core

OpenCode CLI with flexible model selection. Commands and templates copied into project via arckit init.

Limitations: No agents, hooks, or bundled MCP servers. Manual reinstall for updates.

Installation: arckit init my-project --ai opencode

All 53 commands produce identical outputs across AI systems. Claude Code provides the richest experience with agents that isolate research context, hooks that automate governance (filename enforcement, context injection, output validation, secret protection), and bundled MCP servers for authoritative cloud documentation. Gemini CLI offers a strong zero-config experience. Codex and OpenCode provide core command functionality.
Commands vs. skills

ArcKit uses slash commands for formal document generation and a skill for conversational guidance:

  • Commands (/arckit:requirements, /arckit:wardley, etc.) — Generate heavyweight governance documents with document control, project integration, and compliance sections. Deliberate invocation prevents accidental triggering.
  • Wardley Mapping skill — Activates automatically for casual questions about evolution stages, doctrine maturity, build vs. buy decisions, and interactive map creation. No formal document output.
  • Dependency ordering — Commands follow a deliberate sequence (principles → stakeholders → requirements → data-model → etc.). The skill is standalone and order-independent.
  • Best of both worlds — Claude Code also exposes commands as skills automatically, so users get explicit /arckit:requirements invocation AND natural language matching when Claude recognises intent.

Getting Started (5 Minutes)

1. Install Claude Code

macOS & Linux (Homebrew):

brew install --cask claude-code

macOS, Linux & WSL (Shell script):

curl -fsSL https://claude.ai/install.sh | bash

Windows (PowerShell):

irm https://claude.ai/install.ps1 | iex

2. Install ArcKit Plugin

/plugin marketplace add tractorjuice/arc-kit

Then install from the Discover tab.

3. Open Your Project

cd my-project
claude

4. Get Oriented

/arckit.start

Shows project status, connected tools, command decision tree, and recommended next steps.

5. Initialize Project Structure

/arckit.init

Creates the projects/ directory structure for architecture artifacts.

6. Follow GDS Phases

Discovery

/arckit.stakeholders Analyze stakeholders for appointment system
/arckit.risk Create risk register
/arckit.sobc Generate Strategic Outline Business Case

Alpha

/arckit.requirements Define requirements
/arckit.principles Establish architecture principles
/arckit.wardley Create Wardley Map for build vs buy decisions

Beta

/arckit.hld-review Review High-Level Design
/arckit.diagram Create C4 architecture diagrams
/arckit.analyze Comprehensive gap analysis
! Warning AI generates first drafts following templates. Always review, validate, and refine AI output with human expertise.
